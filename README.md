# tutorial-blog-writing

一套用于 AI 生成中文技术教程/博客文章的写作规范。核心目标：**让一个完全零基础的小白，读完文章后能成为这个领域的专家**。

---

## 为什么需要这个 Skill？

AI 生成技术文章有一类通病：

- 开头必是"在当今这个 xxx 的时代..."
- 结尾必是"希望这篇文章对你有所帮助"
- 注释写得像教科书，整整齐齐对齐
- 大段大段的纯文字，代码只是个点缀
- 术语不解释，默认读者"应该懂"
- 热情洋溢但毫无干货

一句话总结：**一眼就能看出是 AI 写的，读不下去。**

这个 Skill 就是解决这个问题的——把 AI 腔洗掉，让文章读起来像真人笔记。

---

## 使用这个 Skill 的好处

| 问题 | 没有 Skill | 有 Skill |
|------|-----------|----------|
| AI 腔 | 文章充满"首先让我们来了解一下..."等教学腔 | 像真人笔记，干净利落 |
| 注释 | 代码注释工整对齐，一看就假 | 注释随意自然，用 `# 解释` 格式 |
| 术语 | 专业词直接甩出来不解释 | 每个术语在首次出现时说人话 |
| 代码块 | 裸 ``` 无语言标注 | 统一加语言标识 |
| 空话 | 大量抒情和套话 | 干货为主，开篇即正题 |
| 目标读者 | 定位模糊 | 明确定位零基础小白，可贯通到专家 |

## 效果对比

### 使用前

```markdown
在当今这个 AI 技术飞速发展的时代，Python 已经成为不可或缺的编程语言之一。
接下来让我们一起来了解一下 Python 列表的基本用法吧。

list = [1, 2, 3, 4, 5]        # 定义一个列表

max_value = max(list)          # 获取最大值

综上所述，列表是 Python 中非常重要的数据结构。希望这篇文章对你有所帮助！
```

### 使用后

```markdown
Python 中，列表（List）是一组有序数据的集合，可以装任何类型的东西。

- 创建一个列表

`number = [1, 2, 3, 4, 5]`

- 获取列表长度

`len(number)`

运行结果

`5`

- 获取最大/最小值

`max(number) # 最大值
min(number) # 最小值`

`number.append(6) # 现在 number 变成了 [1,2,3,4,5,6]`

`number.pop(1) # 删除第 1 个元素（值为 2 的那个）`
```

## 怎么用？

### 方式一：直接在对话中加载

在支持 Skill 的 AI 编程助手中，直接说：

> 用 tutorial-blog-writing skill 帮我写一篇关于 XXX 的教程

AI 会自动加载这套写作规范来生成文章。

### 方式二：作为提示词模板

把 `tutorial-blog-writing/SKILLS.md` 的内容粘贴到任意 AI 对话中作为 system prompt 约束，然后提写作需求。

### 方式三：手动对照

写作或审稿时，对照 `tutorial-blog-writing/SKILLS.md` 末尾的**自检清单**逐条检查：

- [x] 用了「笔者」自称？
- [x] emoji 在 2-5 个？
- [x] 代码块都标注了语言？
- [x] 注释格式是 `code  # comment`？
- [x] 术语都解释清楚了？
- [x] 没有客套话？
